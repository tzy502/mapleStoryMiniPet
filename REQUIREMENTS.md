# 需求文档 — MapleStory MiniPet v0.3

> 桌面悬浮宠物（冒险岛风格），运行于 macOS 14+，Swift 5.9+ / AppKit / NSPanel。  
> 展示冒险岛怪物/NPC 动画，通过 Hermes 人格 (`pet-chat`) 赋予宠物「灵魂」。

---

## 一、项目概述

### 1.1 定位
一款 macOS 桌面悬浮窗宠物应用。宠物以冒险岛怪物/NPC 精灵图动画呈现，行为由 Hermes Agent 实时驱动，具备完整的感知-表达-交互闭环。

### 1.2 核心闭环

```
Hermes Agent (pet-chat 人格)
    │
    ├─ 感知 ─→ Hermes 状态 (idle/working/done/error)
    │         系统事件 (时间、CPU、通知)
    │
    ├─ 决策 ─→ 动画切换 (stand/move/attack/skill/fly)
    │         表情表达 (开心/疑惑/生气/睡觉)
    │
    └─ 表达 ─→ 聊天气泡 (游戏内聊天戒指风格)
               终端输出 (游戏 UI 风格终端)
```

### 1.3 关键概念

| 概念 | 说明 |
|------|------|
| **pet-chat** | 桌宠的 Hermes 人格配置，定义宠物的性格、语言风格、知识边界 |
| **聊天戒指气泡** | 冒险岛游戏内聊天框 UI 风格的气泡，用于宠物发言 |
| **纸娃娃** | 冒险岛角色换装系统，player 的多层装备叠加渲染 |
| **地图背景** | 冒险岛游戏地图（射手村等），作为宠物窗口背景 |
| **椅子/骑宠** | 冒险岛椅子（坐姿）+ 坐骑（骑乘），宠物可交互的姿态道具 |
| **WZ origin** | 冒险岛资源文件中的原始锚点，用于精灵图对齐避免抖动 |

### F2 — 宠物发言系统（pet-chat 人格集成）

**目标**: 宠物拥有独立人格 (`pet-chat`)，通过 Hermes 获得「说话」能力，在桌面以冒险岛风格的聊天气泡展示。

#### F2.1 pet-chat 人格定义

pet-chat 是桌宠专用的 Hermes 人格配置，决定宠物的：

| 维度 | 设计 |
|------|------|
| **性格基调** | 傲娇 / 吐槽 / 可爱 / 毒舌（可配置切换） |
| **知识边界** | 仅了解冒险岛世界、桌宠自己、用户（"胶水"）；不主动讨论现实世界话题 |
| **发言风格** | 短句为主（≤30 字），带颜文字和语气词，模拟游戏 NPC 对话 |
| **触发逻辑** | 随机闲话 / 响应用户操作 / Hermes 状态变化 / 定时问候 |
| **沉默策略** | 大部分时间不说话，避免打扰。发言频率：每小时 3~5 次 + 事件触发 |

#### F2.2 聊天气泡 UI

- 使用冒险岛游戏内「聊天戒指」气泡风格
- 气泡位置：宠物头顶上方，自适应文本宽度
- 支持多行，最多 3 行
- 淡入/淡出动画，3~5 秒后自动消失
- 气泡不可交互（纯展示），不阻挡拖拽

#### F2.3 Hermes 集成

```
MiniPet  ←→  Hermes Client (pet-chat 人格)
  │              │
  │  事件推送     │  Hermes 状态变化
  │  用户交互     │  Hermes 输出 → 过滤 → 气泡展示
  │  (拖拽/右键)  │  
```

- MiniPet 启动时自动创建 Hermes 子会话，加载 `pet-chat` 人格
- 宠物事件（被拖拽、右键、空闲太久）通过 stdin 注入 Hermes
- Hermes 输出经过过滤器（只保留适合气泡的短句），渲染到聊天气泡
- 终端（TerminalView）仍然运行用户的主 Hermes 会话，两者独立

**验收标准**:
- [ ] pet-chat 人格可配置，支持多种性格切换
- [ ] 气泡样式为冒险岛聊天戒指风格
- [ ] 发言频率合理，不打扰用户
- [ ] 拖拽宠物时气泡不跟随移动（或自动收起）

### F4 — Hermes 状态感知与动作反馈

**目标**: 宠物实时感知 Hermes Agent 的运行状态，自动切换对应动画。

#### F4.1 状态映射

| Hermes 状态 | 宠物动画 | 说明 |
|-------------|---------|------|
| `idle` | stand (循环) | Hermes 空闲，宠物站立 |
| `thinking` / `running_tool` | move (循环) | Hermes 工作中，宠物走动/思考 |
| `replying` | skill (单次) | Hermes 即将输出回复 |
| `error` | hit (单次) | Hermes 出错，宠物受击 |
| `done` | happy (单次) | 任务完成，宠物开心 |
| `user_active` | alert (单次) | 用户回来了，宠物注意到 |

#### F4.2 状态获取

- **轮询方式**: `senseHermes()` 每 2s 检查一次 Hermes 状态（现有方式优化）
- **事件方式**（推荐）: Hermes 通过 WebSocket/pipe 主动推送状态变化
- 状态缓存：避免重复切换同一动画

**验收标准**:
- [ ] 6 种状态映射全部实现，动画衔接无卡顿
- [ ] 状态切换后，oneShot 动画播完回 stand，loop 动画持续到状态结束
- [ ] 状态检测延迟 < 3s

### F6 — 背景系统：冒险岛地图

**目标**: 导入冒险岛游戏地图作为宠物窗口背景，宠物「站在地图上」。

#### F6.1 地图资源

| 维度 | 详情 |
|------|------|
| 数据来源 | WZ 中的 Map 资源，路径 `Map/Map{code}.img` 或 `Map/Back/{code}.img` |
| 数据获取 | `wiki-backend` → `POST /api/wz/renderImgPath` + 逐帧下载 → 拼接全图 |
| 地图类型 | 站立图（Back）、地面图层（Tile）、远景（Obj）等，按层级合成 |
| 常用地图 | 射手村、魔法密林、废弃都市、天空之城、玩具城等经典场景 |

#### F6.2 渲染方式

| 维度 | 详情 |
|------|------|
| 图层合成 | 远景 → 中景 → 地面 → 前景，宠物在「地面」层之上 |
| 窗口适配 | 窗口大小 = 地图尺寸（或按比例缩放），最小 312×355（宠物大小） |
| 地图切换 | 右键菜单 → 背景 → 地图列表 → 选择地图 |
| 默认地图 | 透明背景（无地图），保持轻量 |
| 缓存策略 | `~/Library/Caches/MiniPet/maps/{mapId}/` 存放地图 PNG + 元数据 |

#### F6.3 缩放与滚动

| 功能 | 说明 |
|------|------|
| 缩放 | 支持 0.5x / 1x / 2x 缩放显示（窗口随之调整） |
| 宠物定位 | 宠物固定在地图中心偏下（站立在地面上），地图缩放时宠物大小不变 |
| 可选视差 | 多层地图按不同速率移动，营造景深感 |

**验收标准**:
- [ ] 可加载至少 5 张冒险岛经典地图作为背景
- [ ] 地图正确分层渲染，宠物站在地面上
- [ ] 支持缩放，宠物位置不偏移
- [ ] 地图缓存复用，不重复下载
- [ ] 透明通道正确，NSPanel 正常工作

### F7 — 椅子与骑宠系统

**目标**: 宠物可使用冒险岛游戏内的椅子和骑宠（坐骑），切换坐姿/骑乘形态。

#### F7.1 椅子

| 维度 | 详情 |
|------|------|
| 数据来源 | WZ 中的 `Install/_Canvas/{code}.img` 椅子资源 |
| 渲染方式 | 椅子图层在下，宠物图层在上，组合渲染 |
| 动画 | 坐姿动画（sit），坐在椅子上时禁用 move/fly/drag |
| 椅子列表 | 右键菜单 → 椅子 → 种类/分类展示 |
| 取消 | 右键 → 椅子 → 「站起」恢复站立 |

#### F7.2 骑宠（坐骑）

| 维度 | 详情 |
|------|------|
| 数据来源 | WZ 中的骑宠资源（`Character` 或 `Item` 分类下的坐骑动画） |
| 骑乘形态 | 宠物图层叠加在骑宠图层之上，组合渲染 |
| 动画 | 骑乘姿态（ride-stand / ride-move），骑宠有独立奔跑动画 |
| 拖拽兼容 | 骑宠状态下支持拖拽移动，使用 ride-move 动画 |
| 切换逻辑 | 椅子与骑宠互斥（不能同时坐椅子又骑坐骑） |

#### F7.3 通用约束

- 椅子/骑宠资源走与 MOB/NPC 相同的缓存路径
- 右键菜单中「椅子」和「骑宠」为同级子菜单
- 切换椅子/骑宠时播放切换过渡动画（淡入淡出）

**验收标准**:
- [ ] 支持 ≥5 种冒险岛椅子
- [ ] 支持 ≥5 种冒险岛骑宠
- [ ] 椅子状态下宠物不可拖拽，骑宠状态下可拖拽
- [ ] 椅子/骑宠随宠物窗口移动，缩放正确

### F10 — R2 Lua 导出数据流程

**目标**: 利用 WzComparerR2 的 Lua 脚本功能导出精灵图数据，建立自动化数据管道。

| 维度 | 详情 |
|------|------|
| 输入 | WZ 文件路径 + 怪物/NPC ID |
| 流程 | R2 Lua 脚本 → 导出帧 PNG + 元数据 (origin, delay, action) → 本地目录 |
| 元数据格式 | JSON: `{"actions": {"stand": [{"frame": 0, "delay": 120, "origin": [0, 0]}, ...]}}` |
| 生成目标 | 直接产出可用于 MiniPet 的精灵图条 + pet_config.json |
| 工具链 | `export_lua/` 目录 → Lua 脚本 + Python 后处理 |

**与 wiki-backend 的关系**:  
- R2 Lua 导出是**离线方案**，适合无后端环境或批量处理  
- wiki-backend 是**在线方案**，适合实时查询和缓存  
- 两种方案可共存，优先使用缓存，缓存未命中时选择在线/离线

**验收标准**:
- [ ] Lua 脚本可独立运行，产出帧 PNG
- [ ] Python 后处理生成标准精灵图条 + pet_config.json
- [ ] 产出结果与 wiki-backend 格式兼容

### F12 — 聊天气泡系统（气球渲染与集成）

**目标**: 通过冒险岛 WZ 聊天气泡（九宫格拼接）展示宠物发言，支持右键菜单切换气球样式。

| 维度 | 详情 |
|------|------|
| **气球资源** | `UI/ChatBalloon.img/{id}` 共 486 种气球样式 |
| **默认样式** | balloonId=560（聊天戒指风格），`clr` 默认 -1（白色字体） |
| **渲染方式** | 原生 Swift `ChatBalloonView` + `NSView.draw()` 九宫格拼接 |
| **数据流向** | 桌宠 (`APIClient.fetchBalloonTiles`) → wiki-backend (`/api/wz/image`) → 本地缓存 |
| **缓存策略** | 首次加载后缓存 11 张切片 PNG（nw/n/head/ne/w/c/e/sw/s/arrow/se），后续免 API |
| **菜单入口** | 右键 → 聊天气泡 → 测试气泡 / 隐藏气泡 / 切换气球样式 |
| **气泡位置** | `ContainerView` 中宠物头顶上方居中，自适应文本宽度 |
| **动画** | 淡入 0.2s → 显示 3~5s → 淡出 0.3s 自动消失 |
| **clr 读取** | `POST /api/wz/property` 不可用时默认 `-1`（白色），后续通过设置页面自定义 |
| **后端交互** | 所有 wiki 后端通信通过 `APIClient`（Swift），不走独立 Python 脚本 |

**与 bot 的关系**:
- 气泡是宠物「说话」的视觉载体
- bot（QQ Bot）负责与 hermes pet-chat 人格交互产生文本
- MiniPet `BotClient` 接收 bot 回复 → 展示到气泡

**验收标准**:
- [ ] 右键菜单可测试气泡显示（预置文本）
- [ ] 气泡样式可切换（循环切换气球 ID）
- [ ] 气泡淡入淡出动画流畅
- [ ] 气球切片缓存有效，断网可复用

### F13 — QQ Bot 集成（BotClient）

**目标**: 通过外部 QQ Bot 框架实现 hermes pet-chat 人格的长连接，MiniPet 只负责展示回复。

| 维度 | 详情 |
|------|------|
| **通信方式** | HTTP API（MiniPet 发请求 → Bot 调 hermes → 返回回复） |
| **事件触发** | 被拖拽、空闲太久、Hermes 状态变化、定时闲话 |
| **协议定义** | `POST {bot_url}/api/send` → `{"message": "...", "source": "minipet"}` |
| **Bot 侧** | 已有 QQ Bot 框架（go-cqhttp/Lagrange/NoneBot 等），hermes 以 pet-chat 人格运行 |
| **MiniPet 侧** | `BotClient`（已定义接口）+ `ChatBalloonView` 展示回复 |
| **数据流** | `PetView` 事件 → `BotClient.speak()` → QQ Bot → hermes → QQ Bot → `BotClient` 回调 → `ChatBalloonView.show()` |

**为什么用 Bot 而不是直接调 hermes CLI**:
- hermes CLI 每次起新 Process，延迟高、资源浪费
- QQ Bot 框架提供长连接，hermes 会话持续存活
- pet-chat 人格可在 QQ 群中独立调试验证

## 五、pet-chat 人格设计附录

> pet-chat 是 MiniPet 的「灵魂」— 一个通过 Hermes Agent 运行的人格配置。

### 人格模板

```yaml
# pet-chat 人格配置 (供 Hermes 加载)
name: pet-chat
description: 冒险岛桌宠人格 — 傲娇怪物的日常吐槽
personality:
  tone: tsundere         # 傲娇 | cute | sarcastic | lazy
  knowledge: maplestory  # 知识域限定
  speaking:
    max_length: 30       # 最大字数
    emoji_style: kaomoji # (｀・ω・´) 风格颜文字
    triggers:
      random_interval: [900, 1800]  # 随机发言间隔(秒)
      on_drag: true                 # 被拖拽时吐槽
      on_idle_long: true            # 空闲太久时卖萌
      on_hermes_busy: true          # Hermes 忙碌时加油
      on_user_back: true            # 用户回来时打招呼
```

### 示例对话

| 场景 | pet-chat 发言 |
|------|-------------|
| 被拖拽 | 「喂喂！别拽我啊 (｀Д´)」 |
| 用户离开 30min 回来 | 「胶水你终于回来了...人家才没有想你呢 (。-`ω´-)」 |
| Hermes 报错 | 「啊！胶水出 bug 了！赶紧修啊喂！」 |
| 随机闲话 | 「今天的冒险岛也很和平呢...」 |
